# JeolgamAI 기능 요구사항 (MVP) v1

## 1. 문서 목적
| 항목 | 내용 |
|---|---|
| 목적 | 팀 승인 및 개발 착수용 기능 요구사항 확정 |
| 대상 | PM, FE/BE/인프라 엔지니어, 리뷰어 |
| 범위 | Internal Beta (8주/22 working days 기준) |
| 산출물 연계 | PRD, 기술설계, API 명세, 테스트 계획 |

## 2. 제품 목표
| 구분 | 정의 |
|---|---|
| 핵심 목표 | AWS/K8s/Prometheus 데이터를 기반으로 비용 최적화 권고를 제시하고 실행 가능한 가이드 제공 |
| 핵심 가치 | 분석 → 설득(리포트/챗) → 실행(가이드) 루프 완성 |
| 1순위 KPI | 권고 채택률(승인 완료 권고 / 전체 권고), 월 추정 절감액 |
| 베타 목표 | 동작하는 베타 제품 + 문서 패키지 완성 (운영 이전 단계) |

## 3. MVP 범위
| 구분 | 포함 | 제외/후속 |
|---|---|---|
| 연동 | AWS, Kubernetes, Prometheus | Slack 등 외부 채널 |
| 분석 | 컴퓨트/스토리지/DB/네트워크/EKS 핵심 룰 동시 커버 | 고급 ML 기반 예측 고도화 |
| 실행 | 실행/롤백 커맨드, 실행 가이드, 권고 승인 기록 | 실제 인프라 자동 변경/자동 롤백 |
| 리포트 | 경영 요약 + 실행 상세, PDF+JSON | 추가 포맷 확장 |
| 챗 | 웹앱 내 컨텍스트 기반 대화형 에이전트 | 외부 채널 챗 연동 |
| 시각화 | Cloudcraft 추천 아키텍처(성공 시) | Cloudcraft 장애 시 대체 렌더러 |

## 4. 사용자 및 권한
| 역할 | 권한 |
|---|---|
| System Admin | 기본 고객 데이터 비가시, 요청 시 임시 접근(감사로그 필수) |
| Company Admin | 연동 관리, 권고 승인/반려, 리포트 관리 |
| Company Operator | 조회, 실행가이드/명령 조회, 승인 불가 |

## 5. 데이터 소스/인증 요구사항
| 소스 | 필수 여부 | 인증 방식 | 비고 |
|---|---|---|---|
| AWS | 선택(셋 중 하나로 시작 가능) | Cross-account IAM Role 기본 + Access Key fallback | 비용 산정의 1차 기준은 Cost Explorer/CUR |
| Kubernetes | 선택 | Read-only ServiceAccount + Token | request/limit/usage 분석 |
| Prometheus | 선택 | Read-only Token | 사용률/오류율/지연시간 기반 근거 |

## 6. 점수 산정 요구사항

### 6.1 점수 프레임
| 항목 | 값 |

| 재현성 | 룰 버전 고정 + 분석 스냅샷 잠금 |
| 감점 공개 | 룰별 감점 내역 전부 공개 |
| 신뢰도 표기 | 퍼센트 수치 표시 |

### 6.2 등급 기준

### 6.3 감점/신뢰도 정책

|---|---|

| 점수 가중 성향 | 비용/신뢰성/보안 강화형 |

## 7. 권고 엔진 요구사항

### 7.1 공통 출력 포맷
| 필수 필드 | 설명 |
|---|---|
| 근거(Evidence) | 메트릭/비용 데이터 기반 근거 |
| 실행 명령 | CLI/콘솔 적용 명령 |
| 롤백 명령 | 변경 실패/이슈 시 복구 절차 |
| 예상 절감액 | 월/연 추정 절감 금액 |
| 리스크 | Low/Medium/High/Critical |
| 출처 | 룰ID + 원칙명 + AWS 문서 링크 |

### 7.2 권고 상태/정렬
| 항목 | 기준 |
|---|---|
| 상태 흐름 | Draft → Reviewed → Approved/Rejected |
| 승인 단위 | 권고 항목 단위 |
| 기본 정렬 | (절감액 × 신뢰도) 내림차순 |

### 7.3 도메인별 필수 룰
| 도메인 | 필수 룰 세트 |
|---|---|
| Compute | 저활용/과대스펙/스케줄오프/RI-SP 후보 |
| Storage | S3 Lifecycle 전구간 + EBS 타입/미연결 + 스냅샷 정리 |
| DB | RDS 사이징/스토리지 최적화/예약 권고 |
| Network | NAT 및 데이터 전송비 경로 최적화 |
| EKS/K8s | request-limit-usage, pod/node 상태, 과대 프로비저닝 |
| Prometheus 연계 | 사용률 + 오류율 + 지연시간 분석 |

## 8. 온프레미스 지원 요구사항
| 항목 | 기준 |
|---|---|
| 지원 범위 | K8s + Prometheus 기반 권고 |

## 9. 화면/UX 요구사항
| 화면 | 필수 구성 |
|---|---|
| 대시보드 | 총점, 필러 점수, 상위 권고, 절감 추이, 리스크 알림 |
| 인프라 상세 분석 | 리소스/상태/사용률/비용/권고/리스크 테이블 |
| 리포트 생성 | 템플릿 선택, ROI 시뮬레이션, PDF 프리뷰 |
| 실행 가이드 | 사전점검/실행/검증/롤백 4단계 |
| 챗 | 실행형 요약 + 펼치기 설명, 권고 고정 대화 |
| 연동 위저드 | 3단계(입력 → 검증 → 샘플 분석 실행) |

## 10. API 요구사항
| 메서드 | 경로 | 목적 |
|---|---|---|
| POST | /api/integrations/aws | AWS 연동 등록/검증 |
| POST | /api/integrations/k8s | K8s 연동 등록/검증 |
| POST | /api/integrations/prometheus | Prometheus 연동 등록/검증 |
| POST | /api/analysis/run | 분석 실행 트리거 |
| GET | /api/analysis/{analysisId} | 분석 결과 조회 |
| POST | /api/recommendations/{id}/approve | 권고 승인/반려 |
| POST | /api/reports/generate | 리포트 생성 |
| POST | /api/chat | 컨텍스트 기반 질의응답 |
| POST | /api/cloudcraft/recommendation-diagram | 추천 아키텍처 다이어그램 생성 |
